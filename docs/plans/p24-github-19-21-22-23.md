## 사전 조사 결과

### #23 FTS5 중복
- `delete_session_full()`이 turns, turn_vectors는 삭제하지만 `turns_fts`는 안 지움
- `INSERT OR IGNORE`로 turns에 중복 없지만 FTS5는 `OR IGNORE` 없이 항상 INSERT

### #22 compact turn 누락
- 파서(python 시뮬레이션) 490 user turns 수집 → DB에 1개만 존재
- `turns` 테이블: `UNIQUE(session_id, turn_index)` 제약
- 파서가 `turn.index = turns.len() as u32`로 순차 부여하므로 UNIQUE 충돌 자체는 아님
- **유력 원인**: `insert_session`이 `INSERT OR IGNORE`이므로, 세션이 이미 존재하면 turns INSERT도 `OR IGNORE`로 기존 행과 충돌 → 조용히 무시
- `--force` 시에도 `delete_session_full`이 FTS 미삭제 → 불완전 정리 후 재삽입 시 중복/누락

### #21 --local-only
- 코드 주석에 "push만 생략, pull은 항상 수행" 의도 명시
- help 텍스트는 "Skip git pull/push" → 불일치
- 사용자가 기대하는 동작: pull + push 모두 스킵

### #19 wiki --since
- `get_sessions_since()`: SQL `start_time >= ?1` 문자열 비교
- `since_date`: `"2026-04-10"`, `start_time`: `"2026-04-10T09:00:00+09:00"`
- 문자열 비교에서 `"2026-04-10" < "2026-04-10T..."` 이므로 동작은 해야 하지만,
  실제로 `"2026-04-09T15:00:00Z"` (UTC) vs `"2026-04-10"` 비교 시 UTC 기준 전날 세션도 포함됨
