# Task 02 — #21 --local-only pull 스킵 + help 수정

## Changed files

- `crates/secall/src/commands/sync.rs`
  - 줄 42~48: `local_only` 시 pull 단계도 스킵하도록 조건 추가
- ~~`crates/secall/src/main.rs`~~ — help 텍스트가 이미 `"Skip git pull/push"`로 올바르므로 변경 불필요

새로 생성되는 파일 없음.

## Change description

### 문제 상세

현재 동작:
```
--local-only → push 스킵, pull은 항상 수행
```

Help 텍스트:
```
--local-only       Skip git pull/push (local-only reindex + ingest)
```

Help은 pull/push 둘 다 스킵이라고 하지만 실제로 pull은 수행됨.

### 수정 단계

#### Step 1 — `sync.rs` pull 단계에 `local_only` 체크 추가

현재 코드 (줄 42~48):
```rust
// local_only는 push만 생략 — pull은 항상 수행하여 다른 기기 세션을 수신
if vault_git.is_git_repo() {
    // pull 로직...
```

변경 후:
```rust
// local_only이면 pull/push 모두 생략
if !local_only && vault_git.is_git_repo() {
    // pull 로직...
```

주석도 변경:
```rust
// local_only이면 git pull/push 모두 생략 — 로컬 reindex + ingest만 수행
```

#### Step 2 — `main.rs` help 텍스트 확인

현재 help 텍스트가 이미 `"Skip git pull/push"`이므로 동작만 맞추면 됨.
코드 동작이 help과 일치하는지 재확인.

  하지만 이슈 #21에서 사용자 기대는 pull도 스킵이므로, help 텍스트에 맞춤.
