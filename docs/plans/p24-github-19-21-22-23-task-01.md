## Change description

### 문제 상세

`delete_session_full()`의 현재 동작:
```rust
pub fn delete_session_full(&self, session_id: &str) -> Result<()> {
    self.delete_session_vectors(session_id)?;  // turn_vectors 삭제
    // DELETE FROM turns WHERE session_id = ?1   ← turns 삭제
    // DELETE FROM sessions WHERE id = ?1        ← sessions 삭제
    // ❌ turns_fts 미삭제
}
```

`--force` reingest 시:
1. `delete_session_full()` → turns/sessions 삭제하지만 FTS5 잔존
2. `insert_session()` → 새 세션 INSERT
3. `insert_fts()` → FTS5에 중복 행 추가

### 수정 단계

#### Step 1 — `delete_session_full()`에 FTS5 DELETE 추가

```rust
pub fn delete_session_full(&self, session_id: &str) -> Result<()> {
    self.delete_session_vectors(session_id)?;
    // FTS5 행 삭제 (turns 삭제 전에 수행)
    self.conn().execute(
        "DELETE FROM turns_fts WHERE session_id = ?1",
        rusqlite::params![session_id],
    )?;
    self.conn().execute(
        "DELETE FROM turns WHERE session_id = ?1",
        rusqlite::params![session_id],
    )?;
    self.conn().execute(
        "DELETE FROM sessions WHERE id = ?1",
        rusqlite::params![session_id],
    )?;
    Ok(())
}
```

## Risks

- FTS5 virtual table에서 `DELETE FROM turns_fts WHERE session_id = ?1`이 정상 동작하는지 확인 필요.
  FTS5는 일반 테이블과 달리 DELETE 구문이 content= 설정에 따라 다를 수 있음.
  현재 `turns_fts`는 content 테이블 없이 standalone이므로 일반 DELETE 사용 가능.
